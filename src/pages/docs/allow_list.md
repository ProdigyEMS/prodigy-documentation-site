This list represents the URLs that your IT department should allow if you are having issues accessing Prodigy.

---

{% callout title="Sites to Allow" %}
In the vast majority of cases, nothing needs to be done to allow Prodigy to function fully. On certain restrictive networks, there may be errors encountered while trying to access Prodigy and take classes. In this case, you can send this list to your IT department to allow access to Prodigy and our content.
{% /callout %}

## Prodigy EMS Core and Subdomains

- \*.prodigyems.com
- \*.prodigyusercontent.com
- Amazon S3 (Prodigy Assets)
- assets-prodigyems.s3.amazonaws.com
- assets-prodigyems.s3-accelerate.amazonaws.com

## Fonts and Styling

- fonts.googleapis.com
- fonts.gstatic.com
- cdnjs.cloudflare.com
- cdn.jsdelivr.net

## Stripe (Payments)

- js.stripe.com

## JW Player (Embedded Video)

- \*.jwplayer.com
- \*.jwpcdn.com
- \*.jwplatform.com
- \*.jwpsrv.com
- \*.jwpltx.com
- jwpsrv-vh.akamaihd.net

## Other Media and External Content

- \*.youtube.com
- youtu.be
- traffic.libsyn.com
- mcdn.podbean.com

{% callout type="warning" title="URLs Not Covered" %}
While this list covers all core and known external services used by ProdigyEMS, additional domains may be required to support: External links embedded in course content, Third-party scripts and APIs, Custom integrations or assets
{% /callout %}
